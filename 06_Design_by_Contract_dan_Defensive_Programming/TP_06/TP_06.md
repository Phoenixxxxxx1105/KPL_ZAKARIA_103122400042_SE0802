**Soal**

Diberikan dua kode yang sama-sama melakukan operasi pembagian. Pertama menggunakan asersi, kedua menggunakan eksepsi.

const assert = require('assert');

function divide(a, b) {
  assert(typeof a === 'number' && typeof b === 'number', 'Nilai harus bilangan bulat');

  assert(b !== 0, 'Tidak bisa pembagian dengan nol');

  return a / b;
}

function divide(a, b) {
  if (typeof a !== "number" || typeof b !== "number") {
    throw new TypeError("Nilai harus bilangan bulat");
  }

  if (b === 0) {
    throw new Error("Tidak bisa pembagian dengan nol");
  }

  return a / b;
}

try {
  const result = divide(10, 2);
  console.log("Hasilnya adalah:", result);
} catch (error) {
  console.error("Error:", error);
}

**Jawaban**

Memilih Antara Asersi dan Eksepsi dalam Defensive Programming

Dalam pengembangan perangkat lunak, menerapkan konsep defensive programming sangat penting untuk memastikan sistem berjalan stabil dan mampu melindungi dirinya dari kesalahan yang tidak terduga. Dua mekanisme utama yang sering digunakan untuk mencapai tujuan ini adalah asersi (assertion) dan eksepsi (exception). Pertanyaan mengenai kapan saat yang tepat untuk menggunakan asersi atau eksepsi, serta apakah kita harus memilih salah satu secara mutlak, merupakan topik fundamental dalam merancang arsitektur kode yang tangguh.

Asersi sebaiknya digunakan sebagai alat bantu utama selama fase pengembangan dan pengujian (development and testing). Fungsi utama asersi adalah untuk memeriksa asumsi internal atau "kontrak" dari seorang pemrogram. Saat kita menulis asersi, kita pada dasarnya sedang menyatakan sebuah kondisi yang harus selalu bernilai benar jika kode kita bekerja sesuai dengan logika yang dirancang. Misalnya, jika sebuah fungsi internal dijamin tidak akan pernah menerima nilai nol dari fungsi internal lainnya dalam satu sistem, kita dapat menggunakan asersi untuk memastikannya. Jika asersi ini gagal, artinya ada bug atau kesalahan logika fatal dalam kode sumber itu sendiri, bukan kesalahan dari sisi pengguna. Oleh karena itu, dalam banyak ekosistem pemrograman, mekanisme asersi sering kali dinonaktifkan saat program dirilis ke tahap produksi (production) agar tidak membebani performa aplikasi.

Di sisi lain, eksepsi didesain untuk menangani situasi yang tidak wajar atau di luar kendali program pada saat runtime (ketika aplikasi sedang benar-benar digunakan). Eksepsi sangat cocok diterapkan untuk memvalidasi input yang berasal dari luar sistem, seperti data yang dimasukkan oleh pengguna akhir, respons dari API jaringan, atau kegagalan saat membaca file. Dalam contoh kode pembagian (divide) yang diberikan pada modul, jika fungsi tersebut dirancang untuk menerima input langsung dari interaksi pengguna, maka menggunakan eksepsi (seperti melempar TypeError atau Error) adalah pilihan yang jauh lebih tepat. Hal ini karena pembagian dengan nol dari sisi pengguna adalah sesuatu yang sangat mungkin dan wajar terjadi, sehingga program harus mampu menangkap (catch) kesalahan tersebut lalu memberikan pesan yang mudah dipahami oleh pengguna tanpa harus mengalami crash atau terhenti secara total.

Menjawab pertanyaan apakah kita harus bergantung sepenuhnya pada asersi atau sepenuhnya pada eksepsi: jawabannya adalah tidak. Keduanya memiliki ruang lingkup dan tujuan yang berbeda, sehingga pendekatan terbaik adalah menggabungkannya secara harmonis. Asersi digunakan untuk menemukan bug di awal dan menegakkan Design by Contract (seperti prekondisi, pascakondisi, dan invariansi) di antara komponen-komponen internal aplikasi selama masa development. Sementara itu, eksepsi digunakan di lapisan terluar aplikasi untuk menangani input eksternal dan kegagalan lingkungan secara aman di tahap produksi. Mengandalkan salah satu saja akan membuat program rentan; hanya memakai asersi bisa membuat program langsung tertutup paksa di tangan pengguna saat terjadi kesalahan input, sedangkan hanya memakai eksepsi untuk setiap pengecekan internal akan membuat kode menjadi berat dan berpotensi menyembunyikan bug logika yang seharusnya diselesaikan langsung oleh developer.
