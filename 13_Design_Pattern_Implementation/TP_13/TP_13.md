## Jawaba

Pada file backend/api/transaksi.php,dan penggunaan design pattern Table-Driven Routing atau Table-Driven Control. Design pattern ini digunakan untuk mengatur alur program berdasarkan nilai action yang dikirim melalui URL.

Berikut potongan kode yang digunakan:

php
$action = $_GET['action'] ?? 'cart';
$method = $_SERVER['REQUEST_METHOD'];

switch ($action) {
    case 'cart':
        showCart();
        break;

    case 'history':
        showHistory();
        break;

    case 'tambah':
        if ($method === 'POST') addToCart();
        else methodNotAllowed();
        break;

    case 'bayar':
        if ($method === 'POST') checkout();
        else methodNotAllowed();
        break;

    case 'batal':
        if ($method === 'POST') clearCart();
        else methodNotAllowed();
        break;

    default:
        http_response_code(404);
        echo json_encode([
            'success' => false,
            'message' => 'Action tidak ditemukan'
        ]);
        break;
}

Nah pada kode di atas, sistem itu mengambil nilai action dari URL yang dikirim oleh pengguna. Setelah itu sistem akan mencocokkan nilai tersebut pada bagian switch-case. Setiap action memiliki fungsi masing-masing, misalnya action cart akan menjalankan fungsi showCart(), action history akan menjalankan showHistory(), dan action bayar akan menjalankan fungsi checkout().

Design pattern ini disebut Table-Driven Routing karena setiap nilai action berfungsi sebagai kunci yang mengarahkan program ke proses yang sesuai. Jadi program tidak perlu menggunakan banyak percabangan if-else yang panjang dan bertingkat.

Keuntungan dari penggunaan design pattern ini adalah kode menjadi lebih rapi dan mudah dipahami. Selain itu, jika di kemudian hari ingin menambahkan fitur baru, misalnya cetak struk, developer hanya perlu menambahkan case baru pada switch-case tanpa mengubah struktur utama program. Dengan begitu proses pengembangan dan pemeliharaan sistem menjadi lebih mudah.
